7 Fizz is a TLS 1.3 implementation.
8 
9 Fizz currently supports TLS 1.3 drafts 28, 26 (both wire-compatible with the
10 final specification), and 23. All major handshake modes are supported, including
11 PSK resumption, early data, client authentication, and HelloRetryRequest.
12 
13 More background and details are available on the
14 [Facebook Code Blog](https://code.fb.com/networking-traffic/deploying-tls-1-3-at-scale-with-fizz-a-performant-open-source-tls-library/).
15 
16 ## Dependencies
17 
18 Fizz largely depends on three libraries: [folly](https://www.github.com/facebook/folly),
19 [OpenSSL](https://www.openssl.org/), and [libsodium](https://github.com/jedisct1/libsodium).
20 
21 ## Source Layout
22 - `fizz/crypto`: Cryptographic primitive implementations (most are wrapping
23  OpenSSL or libsodium)
24 - `fizz/record`: TLS 1.3 record layer parsing
25 - `fizz/protocol`: Common protocol code shared between client and server
26 - `fizz/client`: Client protocol implementation
27 - `fizz/server`: Server protocol implementation
28 - `fizz/tool`: Example CLI app source
29 
30 ## Design
31 
32 The core protocol implementations are in `ClientProtocol` and `ServerProtocol`.
33 `FizzClientContext` and `FizzServerContext` provide configuration options.
34 `FizzClient` and `FizzServer` (which both inherit from `FizzBase`) provide
35 applications with an interface to interact with the state machine.
36 `FizzClient`/`FizzServer` receives events from the application layer, invokes the
37 correct event handler, and invokes the application `ActionVisitor` to process the
38 actions.
39 
40 `AsyncFizzClient` and `AsyncFizzServer` provide implementations of the folly
41 `AsyncTransportWrapper` interface. They own an underlying transport (for example
42 `AsyncSocket`) and perform the TLS handshake and encrypt/decrypt application
43 data.
44 
45 ## Features
46 
47 Fizz has several important features needed from a modern TLS library.
48 
49 ### Performance
50 
51 Fizz supports scatter/gather IO by default via folly's IOBufs, and will encrypt
52 data in-place whenever possible, saving memcpys. Due to this and several
53 other optimizations, we found in our load balancer benchmarks that Fizz has 10%
54 higher throughput than our prior SSL library which uses folly's
55 [AsyncSSLSocket](https://github.com/facebook/folly/blob/master/folly/io/async/AsyncSSLSocket.h).
56 Fizz also consumes less memory per connection than AsyncSSLSocket.
57 
58 ### Async by default
59 
60 Fizz has asynchronous APIs to be able to offload functions like certificate
61 signing and ticket decryption. The API is based on folly's
62 [Futures](https://github.com/facebook/folly/tree/master/folly/futures) for painless
63 async programming.
64 
65 ### TLS features
66 
67 Fizz supports APIs like exported keying material as well as zero-copy APIs
68 needed to use TLS in other protocols like QUIC.
69 
70 ### Secure design abstractions
71 
72 Fizz is built on a custom state machine which uses the power of the C++ type system
73 to treat states and actions as types of their own. As the code changes, this allows us
74 to catch invalid state transitions as compilation errors instead of runtime errors and
75 helps us move fast.

77 ## Sample Applications
78 
79 Fizz includes an example program that showcases the basic client/server functionality
80 supported by Fizz. The binary is called `fizz` and it has similar usage to the
81 `openssl` or `bssl` commands.
82 
83 For example, to start a TLS server on port 443 with a specified cert:
84 ```
85 fizz server -accept 443 -cert foo.pem -key foo.key
86 ```
87 
88 Then, on the same host, you can connect with:
89 
90 ```
91 fizz client -connect localhost:443
92 ```
93 
94 Both ends will echo whatever data they receive and send any terminal input to the
95 peer. Hitting CTRL+D on either end will terminate the connection.
96 
97 The source code for this program can be found under `fizz/tool`.

99 ## Building
100 
101 ### Ubuntu 16.04 LTS
102 
103 To begin, you should install the dependencies we need for build. This largely
104 consists of [folly](https://github.com/facebook/folly)'s dependencies, as well as
105 [libsodium](https://github.com/jedisct1/libsodium).
106 
107 ```
108 sudo apt-get install \
109  g++ \
110  cmake \
111  libboost-all-dev \
112  libevent-dev \
113  libdouble-conversion-dev \
114  libgoogle-glog-dev \
115  libgflags-dev \
116  libiberty-dev \
117  liblz4-dev \
118  liblzma-dev \
119  libsnappy-dev \
120  make \
121  zlib1g-dev \
122  binutils-dev \
123  libjemalloc-dev \
124  libssl-dev \
125  pkg-config \
126  libsodium-dev
127 ```
128 
129 Then, build and install folly:
130 
131 ```
132 git clone https://github.com/facebook/folly
133 mkdir folly/build_ && cd folly/build_
134 cmake ..
135 make -j $(nproc)
136 sudo make install
137 ```
138 
139 And lastly, build and install fizz.
140 
141 ```
142 cd ../..
143 git clone https://github.com/facebookincubator/fizz
144 mkdir fizz/build_ && cd fizz/build_
145 cmake ../fizz
146 make -j $(nproc)
147 sudo make install
148 ```

150 ### Building on Mac
151 
152 The following instructions were tested on MacOS High Sierra
153 with Xcode 9.4.1. They should work with later Xcode versions as well.
154 
155 Run the helper script from within the `fizz` subdirectory. The helper
156 script assumes that you have homebrew installed and are using homebrew
157 as your package manager. To install homebrew use the instructions on
158 the homebrew [website](https://brew.sh/).
159 
160 It will install and link the required dependencies and also build folly.
161 This may take several minutes the first time.
162 
163 ```
164 cd fizz
165 ./mac-build.sh
166 ```
167 
168 After building, the directory `out/` will contain the libraries as well as
169 `out/bin` will contain the `ClientSocket` and `ServerSocket` binaries.
170 Running it again will be faster and only rebuild `fizz`.
171 
172 You can also install both `fizz` as well as `folly` to a custom directory
173 using the build script, by supplying an `INSTALL_PREFIX` env var.
174 
175 ```
176 INSTALL_PREFIX=/usr/local ./mac-build.sh
177 ```
178 
179 You might need to run the script as root to install to certain directories.
